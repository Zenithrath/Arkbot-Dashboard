# ArkBot Dashboard - Feature Roadmap Recommendation

## Current State Summary

**Stack:** React 19 + TypeScript + Vite + Tailwind + shadcn/ui + Supabase + n8n (Easypanel)

**Pages:** Public Chat, Login, Admin Documents, Admin Chat, Admin Upload

**DB Tables:** `drive_file_sync` (sync tracker), `documents` (RAG knowledge base)

**Backend:** n8n webhooks (chat + upload), no code in this repo

**Gaps identified:** No error handling UX, no confirmation dialogs, no pagination, no file preview, code duplication between chat pages, no service abstraction, no tests, no real-time updates

---

## Recommended Improvements (Prioritized)

### PRIORITY 1 - Quick Wins (High Impact, Low Effort)

#### 1. Confirmation Dialog for Delete
- **Why:** Currently `handleDelete` deletes immediately with no confirmation. Dangerous for production.
- **How:** Use existing `AlertDialog` component (already installed in shadcn/ui)
- **Where:** `DocumentsPage.tsx` — wrap delete button with AlertDialog

#### 2. Toast Notifications for All Operations
- **Why:** Delete/upload operations have no user feedback on success/failure
- **How:** Use existing `sonner` toast (already installed, already used in some places)
- **Where:** `DocumentsPage.tsx` (delete success/error), `UploadPage.tsx` (upload success/error)

#### 3. Chat Logic Deduplication
- **Why:** `ChatPage.tsx` and `AdminChatPage.tsx` are ~95% identical (copy-paste). Bug fixes must be applied twice.
- **How:** Extract shared chat logic into `src/hooks/useChat.ts` custom hook
- **Where:** Create `src/hooks/useChat.ts`, simplify both chat pages

#### 4. Document File Preview/Download
- **Why:** Admins can see file names but can't preview or download the actual content
- **How:** Add preview modal (Dialog component) that shows document metadata + link to Google Drive
- **Where:** `DocumentsPage.tsx`

### PRIORITY 2 - Core Improvements (Medium Effort)

#### 5. Pagination for Documents Table
- **Why:** Currently loads ALL documents at once. Will slow down as knowledge base grows.
- **How:** Use `@tanstack/react-table` (already installed) with server-side pagination via Supabase `.range()`
- **Where:** `DocumentsPage.tsx`

#### 6. Activity Log / Audit Trail
- **Why:** No visibility into who deleted what, when files were uploaded, or chat usage patterns
- **How:** Add a simple `activity_log` table in Supabase, log key actions (delete, upload, chat session)
- **Where:** New page `/admin/activity` or add to existing pages

#### 7. Dashboard Overview Page
- **Why:** No central view — admin lands directly on Documents. No stats at a glance.
- **How:** Add `/admin/dashboard` with key metrics: total documents, total files synced, recent activity, chat usage (if tracked)
- **Where:** New `DashboardPage.tsx`, update routing in `App.tsx`

#### 8. Error Boundary
- **Why:** If any component crashes, the whole app dies with a blank screen
- **How:** Add React Error Boundary at app root with a friendly error page
- **Where:** `App.tsx`

### PRIORITY 3 - Nice to Have (Consider Later)

#### 9. User Management
- **Why:** Currently single admin login. No way to add/remove admins or view who's logged in.
- **How:** Supabase already has auth — add a user list page using `supabase.auth.admin.listUsers()` (requires service role key)
- **Where:** New `UsersPage.tsx`, new nav item in sidebar

#### 10. Real-time Sync Status
- **Why:** DocumentsPage shows static data. If n8n syncs a new file, admin must manually refresh.
- **How:** Supabase Realtime subscription on `drive_file_sync` table
- **Where:** `DocumentsPage.tsx` — add `.channel().on('postgres_changes')` subscription

#### 11. Chat History Search/Export
- **Why:** Chat history is only in localStorage. Lost on cache clear. No way to review past conversations.
- **How:** Save chat sessions to Supabase `chat_sessions` table, add search page
- **Where:** New hook + optional admin page

#### 12. Document Search/Chat Context Preview
- **Why:** Can't see what chunks the RAG system has indexed for a document
- **How:** Query `documents` table filtered by `drive_file_id`, show chunks in a modal
- **Where:** `DocumentsPage.tsx` — add "View Chunks" button per row

## What I Recommend NOT Doing (Overkill)

| Idea | Why It's Overkill |
|------|-------------------|
| GraphQL layer | Only 2 tables, Supabase REST is fine |
| State management (Redux/Zustand) | App is simple enough for useState + props |
| Full i18n system | Only 1 language needed (Indonesian), hardcoded is fine |
| Micro-frontend architecture | Single team, single deploy — not needed |
| CI/CD pipeline with tests | Good in theory but adds complexity for a small dashboard |
| WebSocket for real-time chat | Current polling/HTTP approach works fine for chat |

---

## Implementation Order (My Recommendation)

Start with Priority 1 items — they're all small, high-impact changes that can be done in one session:

1. **Confirmation dialog for delete** (AlertDialog) — 15 min
2. **Toast notifications** (sonner) — 15 min  
3. **Extract chat hook** (useChat.ts) — 30 min
4. **Document preview** (Dialog) — 20 min

These 4 changes will make the biggest UX difference with minimal code changes.
