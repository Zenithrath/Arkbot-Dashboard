# Plan: Split Drive Manager dari DocumentsPage

## Goal
Extract Google Drive tab code dari `DocumentsPage.tsx` (1097 baris) ke component terpisah. Documents page tetap punya tab Database + Drive Manager, tapi code-nya terpisah.

## Current Problem
`DocumentsPage.tsx` adalah monolith 1097 baris dengan 26 state variables. Tab Google Drive (list, delete, search) di-mix dengan Database tab logic.

## Approach

### 1. Create `src/components/drive/DriveManager.tsx`
Self-contained component yang handle semua Google Drive tab functionality:
- **State**: `driveFiles`, `driveLoading`, `driveSearch`, `deleteDriveTarget`, `deleteDriveDialogOpen`
- **Functions**: `fetchDriveFiles()`, `handleDeleteDriveFile()`, `formatDate()`
- **UI**: Search bar, files table, delete confirmation dialog
- **Props**: None needed — component is fully self-contained, manages its own data fetching via n8n webhook
- **Auto-fetch**: Fetches files on mount via `useEffect`

Lines extracted from DocumentsPage.tsx:
- Lines 119-169: `fetchDriveFiles`, `handleDeleteDriveFile`
- Lines 774-870: Drive files table UI
- Lines 1066-1094: Delete Drive File Dialog
- Lines 54-60: `DriveCloudFile` interface
- Lines 42-43: `N8N_DRIVE_MANAGER_URL` constant

### 2. Create `src/components/drive/types.ts`
Shared types:
- `DriveCloudFile` interface
- `N8N_DRIVE_MANAGER_URL` constant

### 3. Simplify `DocumentsPage.tsx`
- Remove Drive-related state: `driveFiles`, `driveLoading`, `driveSearch`, `deleteDriveTarget`, `deleteDriveDialogOpen`
- Remove Drive-related functions: `fetchDriveFiles`, `handleDeleteDriveFile`
- Remove Drive-related UI (table + dialog)
- Remove `DriveCloudFile` interface and `N8N_DRIVE_MANAGER_URL`
- Remove `useEffect` for auto-fetching drive files (line 203-207)
- Replace manual tab buttons with `<DriveManager />` component
- Keep `activeTab` state for tab switching
- Keep Database tab + all its logic intact

### 4. Tab Badge Count
DriveManager will expose file count via a simple pattern:
- DriveManager handles its own fetch and renders its own content
- Tab badge in DocumentsPage shows count — pass `onFileCount` callback prop if badge count is needed, OR just keep the DriveManager internal count simple with a small `onCount` prop

**Decision**: Use a simple `onCount?: (count: number) => void` prop on DriveManager so DocumentsPage can show the file count in the tab badge.
